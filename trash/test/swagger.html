Finhub API Swagger v1.2

Documentación oficial de los endpoints protegidos con JWT. Autentica con /auth/login o /auth/validate para obtener cookies HttpOnly y prueba cuentas, usuarios y objetos financieros.

Flujo rápido
  • POST /auth/login o /auth/validate con email y password para emitir cookies de sesión.
  • Swagger envía las cookies automáticamente (credentials: include). Usa el botón "Authorize" si prefieres un header Bearer.
  • Admin: gestiona /users, /financial-objects (POST/PUT/DELETE) y /accounts (crea/actualiza/elimina).
Compat: /accounts/{id}/update y /accounts/{id}/delete aceptan POST para clientes legados.
